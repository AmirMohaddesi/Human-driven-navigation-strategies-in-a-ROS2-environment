Multi-Agent ROS2 Platform Documentation

Welcome to the comprehensive documentation for the Multi-Agent ROS2 Platform, designed specifically for scholars and researchers exploring robotics in disaster response scenarios. Built using ROS2 Humble on Ubuntu 22.04, this platform provides an accessible environment for deploying and managing multiple TurtleBot3 robots for coordinated mapping, navigation, and exploration tasks.

Introduction

Project Overview

This platform allows users to simulate multiple robotic agents operating simultaneously in disaster-response contexts. Using popular robotics software packages, the platform demonstrates autonomous navigation, real-time mapping (SLAM), and effective visualization.

Use Cases and Target Audience

Ideal for scholars from diverse fields who require a robust yet user-friendly robotic simulation platform, this project supports research and experimentation in swarm robotics, coordinated exploration, disaster mapping, and related areas.

Features

  • Multi-Agent Simulation: Deploy multiple TurtleBot3 robots in Gazebo simulations.
  • Real-Time SLAM Mapping: Simultaneous Localization and Mapping using SLAM Toolbox.
  • Autonomous Navigation: Equipped with Navigation2 (Nav2) for autonomous movement.
  • Interactive Visualization: Real-time visualization and interaction through RViZ.

Installation Guide

1. ROS2 Humble Installation

Ensure your system is Ubuntu 22.04. Follow the official ROS2 Humble Installation Guide.

2. Dependency Installation

Execute the following commands:

sudo apt update
sudo apt install ros-humble-desktop ros-humble-navigation2 ros-humble-nav2-bringup ros-humble-slam-toolbox ros-humble-gazebo-* ros-humble-turtlebot3* ros-humble-rviz2

3. Environment Configuration

Run these commands to set environment variables:

echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
echo "export TURTLEBOT3_MODEL=waffle" >> ~/.bashrc
source ~/.bashrc

Troubleshooting Common Issues

  • Gazebo not loading: Ensure environment variables and dependencies are properly installed.
  • RViZ visualization errors: Confirm that your TURTLEBOT3_MODEL variable matches installed models.
